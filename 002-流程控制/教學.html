A if 判斷：分支、比較/邏輯運算子、if 帶初始化

對應檔案：sample-01-if判斷.go ｜ 跑：go run 002-流程控制/sample-01-if判斷.go

1. if / else if / else

最基本的分支。Go 的寫法有兩個和 C/Java 明顯不同的慣例：

  • 條件式不加小括號：寫 if amount >= 999，不是 if (amount >= 999)。
  • 大括號 { } 一定要有（就算只有一行），而且 { 必須跟在同一行結尾（這是 gofmt 強制的）。
if orderAmount >= 999 {
    fee = 0            // 滿額免運
} else {
    fee = 60
}
慣例{ 一定和 if／else 同一行——這不是風格喜好，是 Go 語法規定（換行寫會編譯錯）。所以你在任何 Go 專案看到的分支長相都一樣，省去排版爭論。

2. 多段 else if

由上往下逐條比對，命中一條就結束，不會再往下比。適合把金額切成幾個級距：

if amount >= 3000 {
    // 送 100 購物金
} else if amount >= 1500 {
    // 送 50 購物金
} else {
    // 未達門檻
}
類比一長串 else if 像「一關一關的門檻」，闖過第一道就進去了、後面的門不再檢查。若級距很多，第 6 節的「無條件 switch」會更好讀。

3. 比較 / 邏輯運算子 & 「if 帶初始化語句」

比較 == != > < >= <=；邏輯 &&（且）、||（或）、!（非）。&& 和 || 有短路特性：a && b 若 a 為 false 就不會去算 b。

Go 特有語法：if 條件前可放一個「初始化語句」，用 ; 隔開。這裡宣告的變數只活在這組 if / else 裡（scope 更小、更安全）：

if remain := 999 - orderAmount; remain > 0 {
    fmt.Printf("再買 %d 元免運\n", remain)
}
// 這裡 remain 已消失，再用會 undefined
為什麼重要這正是電商 code 裡到處可見的 if err := doSomething(); err != nil { ... } 的骨架——先執行一件事、順手把結果（常是 err）綁進 if 的 scope，用完即丟。主題 008 錯誤處理會天天用到。

B switch：case 自動 break、多值、switch true

對應檔案：sample-02-switch.go ｜ 跑：go run 002-流程控制/sample-02-switch.go

4. 基本 switch —— case 會「自動 break」

依一個值分支。Go 最大的不同：每個 case 執行完會自動跳出，不用像 C/Java 在每個 case 尾巴寫 break（忘了寫就會不小心「貫穿」到下一個 case，那是 C 的經典 bug）。

switch status {
case Pending:
    fmt.Println("提醒付款")
case Paid:
    fmt.Println("通知揀貨")   // 命中就停，不會漏到下面
default:
    fmt.Println("轉人工")
}
重點反轉C/Java 是「預設貫穿、要停得寫 break」；Go 是「預設停、要貫穿得寫 fallthrough」。Go 選了比較安全、比較符合直覺的預設。

5. 一個 case 列多個值 + default

用逗號把多個值放進同一個 case，命中任一個就執行；default 接住其餘情況（放哪一行都行，慣例放最後）。

switch status {
case Pending, Cancelled:
    fmt.Println("尚未進入出貨流程")
case Paid, Shipped, Completed:
    fmt.Println("已收款，訂單有效")
}

6. 無條件 switch（switch true）—— 取代一長串 else if

switch 後面不接值時，等同 switch true，會逐條檢查「哪個 case 的條件為真」。這是 Go 慣例：級距一多，它比一串 else if 好讀。

switch {
case amount >= 3000:
    fmt.Println("鑽石單")
case amount >= 1500:
    fmt.Println("黃金單")   // 命中就停
default:
    fmt.Println("一般單")
}
慣例看到 switch { case 條件: ... }（switch 後面空空的）別困惑，這就是 Go 版的「if-else if 鏈」，在電商分級、狀態機邏輯裡很常見。
陷阱：fallthrough真想貫穿到下一個 case，得手動寫 fallthrough——它會無條件執行下一個 case（不再檢查那個 case 的條件！），很容易誤用，實務上很少出現。看到它要特別小心。

C for 迴圈：Go 唯一的迴圈，四種形態

對應檔案：sample-03-for迴圈.go ｜ 跑：go run 002-流程控制/sample-03-for迴圈.go
核心觀念Go 只有 for 一個迴圈關鍵字，沒有 while、沒有 do-while。別找了——所有迴圈需求都用 for 的不同寫法達成。

7. for range —— 走訪 slice/map 最常用

range 每輪回傳 (索引, 元素副本)。用不到索引就用 _ 忽略（Go 不允許宣告了卻不用的變數，_ 是「我故意不要」）。

for _, item := range cart {
    totalAmount += item.Price * item.Qty
    totalQty    += item.Qty
}
陷阱：range 給的是「副本」迴圈裡的 item 是元素的複製，改 item.Qty 不會改到 cart 裡的原值。要改原值得用索引 cart[i].Qty = ...。（這牽涉到值/指標，主題 006 會講透。）

8. 三段式 for：init; condition; post

最傳統的計數迴圈，三段用 ; 隔開，需要「第幾項」時用它跑索引：

for i := 0; i < len(cart) && i < 2; i++ {
    fmt.Println(cart[i].Name)   // 列出前 2 項當推薦
}

9. for 當 while 用 / 無限迴圈

把三段式只保留中間的「條件」，就是別的語言的 while；連條件都不寫就是無限迴圈 for {}，靠 break 跳出、continue 跳過本輪。

for amount < 5000 {      // = while (amount < 5000)
    amount += 200
}

for {                    // 無限迴圈
    if done { break }    // 跳出整個迴圈
    if skip { continue } // 跳過本輪、進下一輪
}
慣例需要 while 就寫 for 條件 { }；需要無限迴圈（如 server 一直收請求）就寫 for { } 再靠 break／return 收場。這是最能看出「Go 只有 for」的地方。

D defer：保證執行的收尾、LIFO、參數求值陷阱

對應檔案：sample-04-defer.go ｜ 跑：go run 002-流程控制/sample-04-defer.go

10. defer 的時機：函式 return 之際才執行

defer 會把一個呼叫「登記起來」，等所在函式即將 return 時才執行。它專為「不管後面成功或失敗，最後都得做的收尾」而生——解鎖、關連線、關檔案。

fmt.Println("鎖定庫存")
defer fmt.Println("解鎖庫存")   // 登記，等 return 才跑

fmt.Println("開啟 DB 連線")
defer fmt.Println("關閉 DB 連線")
黃金慣例一「取得資源」就在下一行立刻 defer「釋放」，成對出現、不會忘（例如 f, _ := os.Open(...); defer f.Close()）。就算函式中途 return 或 panic，defer 依然會執行，資源不外漏。

11. 多個 defer 是 LIFO（後進先出）

同一函式登記多個 defer，執行順序是倒過來的——最後登記的最先執行，像疊盤子：最後放上去的最先拿下來。

// 登記順序：解鎖庫存 → 關閉 DB 連線
// 執行順序（return 時）：關閉 DB 連線 → 解鎖庫存
為什麼是 LIFO因為資源常是「層層取得」的（先鎖庫存、再開連線），釋放就該「反向」進行（先關連線、再解鎖），才對得起取得的順序。LIFO 剛好自動幫你做到這件事。

12. 陷阱：defer 的「參數在當下就求值」

最容易踩的雷：defer f(x) 裡的參數 x，是在寫下 defer 那一刻就被求值並「凍結」，不是等到 return 才算。

stage := "下單"
defer fmt.Println("defer 讀到：", stage)  // "下單" 當場被凍結
stage = "出貨"
// return 時仍印「下單」，不是「出貨」！
陷阱別以為 defer 會讀「最後的值」。參數在登記時就複製好了。想讀最新值，改包一層匿名函式（閉包在執行時才讀變數）：defer func(){ fmt.Println(stage) }()。閉包會留到主題 003 詳談。
