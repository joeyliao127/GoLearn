A 多回傳值：一次回兩個結果、具名回傳、(值, error) 慣例

對應檔案：sample-01-多回傳值.go ｜ 跑：go run 003-函式/sample-01-多回傳值.go

1. 一次回傳多個值

和很多語言不同，Go 函式可以回傳不只一個值——不用硬湊成一個 struct 或陣列。呼叫端用「多重指派」一次接住：

func priceBreakdown(unitPrice, qty int) (int, int) {
    subtotal := unitPrice * qty
    tax := subtotal * 5 / 100
    return subtotal, tax
}

subtotal, tax := priceBreakdown(1000, 3)   // 一次接住兩個
慣例用不到某個回傳值時，用 _ 明確丟掉：_, tax := priceBreakdown(...)。Go 不准「宣告了卻不用」的變數，_ 是「我故意不要這個」的官方寫法。
為什麼是內建多回傳值不是花招——它就是為了下面的 (結果, error) 慣例而生：一個位置放結果、一個位置放「有沒有出錯」，Go 因此不需要 try/catch。

2. 具名回傳值（named return values）

可以在回傳型別的地方先幫回傳值取名字。(subtotal, tax int) 等於在函式開頭就宣告好這兩個變數（型別都是 int，初值是零值 0），函式內直接對它們賦值：

func priceBreakdown(unitPrice, qty int) (subtotal, tax int) {
    subtotal = unitPrice * qty      // 直接用、不必再宣告
    tax = subtotal * 5 / 100
    return subtotal, tax            // 仍建議明確寫出來
}
類比具名回傳值像「先在表單上印好欄位名（小計、稅額）」，函式跑的時候只要把數字填進去。好處是讀函式簽名就知道「它回的兩個 int 各是什麼意思」。

3. (值, error) 慣例 —— Go 錯誤處理的骨架

這是 Go 最重要的慣例：把「可能失敗」的操作寫成回傳 (結果, error)；成功時 error 是 nil，失敗時回「零值 + 一個描述錯誤的 error」。呼叫端馬上用 if err != nil 檢查：

func applyCoupon(unitPrice int, code string) (int, error) {
    if code == "SAVE100" {
        return unitPrice - 100, nil          // 成功：結果 + nil
    }
    return 0, errors.New("無效的折扣碼：" + code) // 失敗：零值 + error
}

price, err := applyCoupon(1000, "SAVE100")
if err != nil {
    // 先處理錯誤（記 log／回傳出去），通常就 return，不再往下
} else {
    // 沒錯才用 price
}
為什麼重要Go 沒有 try/catch；錯誤就是一個普通的回傳值。「回 (值, error) → 呼叫端 if err != nil → 早 return」是你之後在每個 Go 專案都會看到的節奏。主題 008 會把 error 的建立、包裝、比對整套講透，這裡先把「長相」記熟。
慣例error 一律放最後一個回傳值；成功回 nil。搭配「if 帶初始化」寫成 if _, err := applyCoupon(...); err != nil { ... }，讓 err 只活在該 if 內（呼應主題 002）。

4. 少用 naked return（裸回傳）

用了具名回傳值後，Go 允許只寫一個光禿禿的 return（不接任何東西），它會自動把「當下那幾個具名回傳值」回傳出去——這叫 naked return：

func priceBreakdown(unitPrice, qty int) (subtotal, tax int) {
    subtotal = unitPrice * qty
    tax = subtotal * 5 / 100
    return          // naked return：自動回 (subtotal, tax)。函式一長就很難讀
}
陷阱／慣例naked return 在稍長的函式裡會讓人「看不出到底回了什麼」，是常見的可讀性地雷。慣例是明確寫出 return subtotal, tax；naked return 只在極短的函式偶爾用。本 repo 一律明寫。

B variadic：收任意多個引數的可變參數

對應檔案：sample-02-variadic.go ｜ 跑：go run 003-函式/sample-02-variadic.go

5. ...T 可變參數 —— 想傳幾個就傳幾個

把最後一個參數寫成 名字 ...T，呼叫時就能帶 0 個、1 個或任意多個 引數（像 fmt.Println 那樣）。在函式內部，這個參數其實就是一個 []T，可以直接 for range 走訪或用 len()：

func sum(prices ...int) int {   // prices 在函式內就是 []int
    total := 0
    for _, p := range prices {
        total += p
    }
    return total
}

sum(2490, 290, 190)   // 傳 3 個 → 2970
sum(999)              // 傳 1 個 → 999
sum()                 // 傳 0 個 → 0（prices 是空 slice）
類比variadic 像結帳台的輸送帶：你要放幾件商品都行，收銀機（函式）在裡面把它們當成「一排東西」（[]int）逐一處理。

6. 用 slice... 把既有的 slice 攤開

如果價格本來就裝在一個 []int 裡，不能直接把整個 slice 當「一個引數」丟給 variadic 參數（型別對不上）。要在後面加 ...，Go 才會把它「攤平」成一個個引數：

cartPrices := []int{1200, 350, 80, 80}
sum(cartPrices...)    // ✅ 把 slice 攤開成 4 個引數
// sum(cartPrices) ❌ 編譯錯：型別是 []int，不是一串 int
慣例兩個規則要記牢：① variadic 參數只能是最後一個（前面可有普通參數，如 func f(name string, items ...int)）；② 傳既有 slice 進 variadic 一定要加 ... 攤開。
陷阱sum(cartPrices)（少了 ...）會編譯失敗；而 append 也吃這招——append(a, b...) 才是把 b 整個接到 a 後面（主題 004 會用到）。

C 閉包：函式是值、能捕捉並記住狀態

對應檔案：sample-03-閉包.go ｜ 跑：go run 003-函式/sample-03-閉包.go
核心觀念在 Go 裡函式是「值」：可以存進變數、當參數傳、當回傳值。當一個函式「用到了它外層的變數」，它就形成閉包（closure）——會把那個外層變數捕捉起來，之後還記得、還能改。

7. 函式是值 & 工廠函式

makeCart() 不直接算錢，而是回傳一個函式。這種「產生函式的函式」很常見（工廠函式）。回傳值型別 func(price int) int 就是「吃一個 int、回一個 int 的函式」：

func makeCart() func(price int) int {
    total := 0
    return func(price int) int {   // 回傳一個匿名函式
        total += price
        return total
    }
}

addToCart := makeCart()   // addToCart 現在是一個函式

8. 捕捉外層變數、記住並改動狀態

上面那個匿名函式用到了外層的 total，於是捕捉了它。關鍵：捕捉的是那個變數本身（by reference），不是它當下的值——所以每次呼叫都在同一個 total 上累加，它「記得」上一次的結果：

addToCart := makeCart()
addToCart(2490)   // 2490
addToCart(290)    // 2780（記得上次的 2490）
addToCart(190)    // 2970

cartA := makeCart()   // 再叫一次 → 全新、獨立的 total
cartB := makeCart()
cartA(100); cartA(100)   // 100 → 200
cartB(500)               // 500（完全不受 A 影響）
類比每呼叫一次 makeCart() 就像「發一台新的計數器」，它把自己的 total 藏在肚子裡（其他人拿不到）。同一台一直加會累積；不同台各記各的。這是閉包「封裝私有狀態」的威力。
接續 002 的伏筆還記得主題 002 說「defer 的參數在當下就凍結，但包一層匿名函式就會讀最新值」嗎？原因就在這裡：閉包捕捉的是變數，在執行時才去讀它，所以讀到的是最新的值。

9. Go 1.22 起：for 迴圈變數「每輪獨立」

一個經典陷阱：在迴圈裡建立多個閉包、各自捕捉迴圈變數。Go 1.21 以前整個迴圈共用同一個變數，所有閉包最後都讀到「最後一輪的值」；Go 1.22 起每一輪的迴圈變數是獨立副本，閉包各自捕捉到正確的值：

names := []string{"鍵盤", "滑鼠", "螢幕"}
var printers []func()
for _, name := range names {
    printers = append(printers, func() { fmt.Println(name) })
}
for _, p := range printers { p() }
// Go 1.22+：依序印 鍵盤 / 滑鼠 / 螢幕
// Go 1.21-：會全印成「螢幕」（都捕捉到同一個 name）
為什麼你多半不用煩惱本 repo 的 go.mod 是 go 1.26.4，早已是新行為，上面的程式碼會印出正確結果。但你在網路上會看到大量「舊寫法」教你 name := name 手動複製一份——那是 1.22 前的解法，現在不需要了。知道有這回事，看老 code 才不會困惑。
