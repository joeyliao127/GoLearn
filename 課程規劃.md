# 課程規劃（002–013 設計 spec）

這份是 **`new-topic` skill 的餵料**：要建某個主題時，讀本檔對應條目 + `CLAUDE.md` 慣例 + 拿 `001-變數與型別/` 當品質範本，就能產出一致的教材與作業。001 已完成、當作活範本，故本檔從 002 起。

## 共用慣例（每個主題都適用）
- 資料夾 `NNN-主題名`；檔案 `sample-NN-<子主題>.go` / `homework-NN-<子主題>.go`，**NN 是該資料夾內從 01 起的流水號**，sample↔homework 同號成對。
- 每個 sample/homework 是自足 `package main` + `//go:build ignore`，`go run <檔>` 單跑（**兩個結構特例見下**）。
- 全部**電商主題**；每個 homework **≤5 分鐘**；一個子主題不夠驗收就拆多個號碼。
- sample 註解逐段對應 `教學.html` 章節；`教學.html` 用四種 callout（key/conv/trap/analogy）、Mermaid、自包含。
- `學習紀錄.html` 六區塊留白模板；建完更新根目錄 `學習總覽.html`（該列 + 路線圖狀態）。

## ⚠️ 兩個結構特例（不套「單檔 //go:build ignore」）
這兩個主題要「用資料夾當 package」（`go run ./dir` / `go test ./dir`），因此**資料夾名與子資料夾名必須 ASCII**——中文會 `malformed import path: invalid char`（import 路徑不收非 ASCII，連 `go build ./...` 都會壞）。做法：scaffold 產中文 folder 後 `mv` 成 ASCII，HTML 標籤仍保留中文。
- **009 套件與慣例** → 資料夾 `009-packages`（已完成，當真 package 範本）：跨包 demo `sample-01-crosspkg/`（main + `pricing/` 子包）；跑 `go run ./009-packages/sample-01-crosspkg`。單檔子主題（命名慣例）仍可中文檔名。
- **013 測試** → 資料夾如 `013-testing`：真 package + `*_test.go`，`go test ./013-testing`。

## 002-流程控制（if/switch、for、defer）
- **目標**：讀懂電商 code 的分支與迴圈；學會 Go 沒有 while、switch 不 fallthrough、defer 資源清理。
- **作業對**：
  - `01 if`：sample=依訂單金額判斷運費（滿 999 免運）；hw=依會員等級 if/else if 給折扣。
  - `02 switch`：sample=依 OrderStatus switch 印處理訊息；hw=依付款方式回傳手續費。
  - `03 for`：sample=for range 購物車加總金額與件數（示範 for 的三種寫法＝Go 的 while）；hw=for 找出最貴商品。
  - `04 defer`：sample=defer 示範 LIFO 與「函式結束才執行」（模擬釋放鎖/關連線的時機）；hw=用 defer 印「訂單處理結束」。
- **idiom/陷阱**：Go 只有 `for`；`switch` case 自動 break、要貫穿才 `fallthrough`；`defer` 後進先出、參數在 defer 當下就求值。
- **教學大綱**：1 if/比較運算 2 switch(無 fallthrough) 3 for 三形態+range 4 defer(LIFO/清理慣例)。

## 003-函式（多回傳值、variadic、閉包）
- **目標**：多回傳值 `(值, error)` 是 Go 的核心慣例；variadic、閉包。
- **作業對**：
  - `01 多回傳值`：sample=`func priceBreakdown(...) (subtotal, tax int)`；hw=回傳 (含稅價, 稅額)。
  - `02 variadic`：sample=`func sum(prices ...int) int` 加總任意商品；hw=`func withTags(name string, tags ...string)`。
  - `03 閉包`：sample=產生「累計購物車金額」的閉包；hw=`makeDiscount(rate)` 回傳折扣函式。
- **idiom/陷阱**：`(結果, error)` 慣例、少用 naked return；variadic 只能是最後參數；閉包捕捉變數（Go 1.22 起迴圈變數每輪獨立）。
- **教學大綱**：1 多回傳值 2 (值,error) 慣例 3 variadic 4 閉包與捕捉。

## 004-集合（slice / map / array）
- **目標**：讀懂電商 domain 的清單與字典；nil 行為與 aliasing 陷阱。
- **作業對**：
  - `01 slice`：sample=購物車 `[]Product` append/走訪/加總；hw=建商品清單、append、算總數。
  - `02 map`：sample=庫存 `map[string]int` 讀寫 + `v, ok :=` 判存在；hw=統計各分類商品數。
  - `03 陷阱`：sample=nil slice 可 append vs nil map 寫入 panic、slice 共享底層陣列的 aliasing；hw=`delete` map key 或避免 aliasing。
- **idiom/陷阱**：`make` vs 字面值；`v, ok := m[k]`；nil slice append OK、nil map 寫入 panic；`len`/`cap`；slice aliasing。
- **教學大綱**：1 slice 2 map 3 array vs slice 4 nil 行為與 aliasing。

## 005-struct與方法（struct、method、receiver、tag、embedding）
- **目標**：讀懂電商 model；方法/receiver；struct tag（接 012）；組合 embedding 取代繼承。
- **作業對**：
  - `01 struct`：sample=`Product`/`Order` 定義、`&T{}` 建立；hw=定義 `Customer` 並建立。
  - `02 方法`：sample=`(o Order) Total() int`（value receiver）；hw=`(p Product) Label() string`。
  - `03 tag`：sample=加 `json:"..."` tag、示範語法；hw=幫 struct 補 json tag。
  - `04 embedding`：sample=`Order` 內嵌 `Audit{CreatedAt}`；hw=用 embedding 共用欄位。
- **idiom/陷阱**：`&T{}` vs `new`；exported 欄位才會被 json 序列化；embedding 是組合不是繼承；receiver 值/指標之爭留到 006。
- **教學大綱**：1 struct/建立 2 方法/receiver 3 struct tag 4 embedding。

## 006-指標（傳值、pointer receiver、new/make）
- **目標**：Go 永遠傳值；何時用指標。**可遷移 `_封存/pointer/main.go` 的訂單範例**。
- **作業對**：
  - `01 傳值vs指標`：sample=`addItemByValue` 改不到 vs `addItemByPointer` 改得到；hw=用指標改訂單狀態。
  - `02 pointer receiver`：sample=`(o *Order) ApplyDiscount(percent)`；hw=幫 struct 寫會改自身的方法。
  - `03 new/make`：sample=`new(T)` vs `&T{}`、`make` slice/map；hw=用 make 建購物車/庫存。
- **idiom/陷阱**：傳指標也 copy（copy 的是地址）；receiver 全值或全指標別混；nil map（呼應 004）。
- **教學大綱**：1 傳值心法 2 `&`/`*` 三符號 3 pointer receiver 4 new/make。
- **銜接**：接 005（方法/receiver）。

## 007-介面（interface、duck typing、DI/mock、any）
- **目標**：讀懂電商 service/repository 的介面骨架；隱式實作；用介面做 DI 與 mock（接 013）。
- **作業對**：
  - `01 介面基礎`：sample=`Notifier` 介面 + Email/SMS 兩實作（duck typing）；hw=`PaymentMethod` 介面兩實作。
  - `02 DI`：sample=`OrderService` 依賴 `OrderRepository` 介面、可換實作；hw=定義 `Repository` 介面 + 假實作。
  - `03 mock`：sample=注入 mock 實作測 service（前導 013）；hw=寫一個 mock。
- **idiom/陷阱**：小介面（常單方法）；介面在「使用方」定義；隱式實作不用 `implements`；nil interface 陷阱。
- **教學大綱**：1 介面與隱式實作 2 duck typing 3 DI 4 mock（+ any/型別斷言簡述）。
- **銜接**：接 005/006。 **（若 4 個點嫌多，any/斷言只在教學帶過、不出獨立作業）**

## 008-錯誤處理（error、wrap %w、errors.Is/As、sentinel、panic/recover）
- **目標**：Go **沒有 try/catch**；error 是值、早 return、包裝與比對。
- **作業對**：
  - `01 error 基礎`：sample=回傳 error + `if err != nil` + `errors.New`/`fmt.Errorf`；hw=`withdraw(balance, amount)` 餘額不足回 error。
  - `02 wrap/Is`：sample=`fmt.Errorf("...: %w", err)` + `errors.Is(err, ErrOutOfStock)`；hw=包裝錯誤 + `errors.Is` 判斷。
  - `03 sentinel/自訂型別`：sample=`var ErrNotFound = errors.New(...)` + 自訂 error struct + `errors.As`；hw=定義 sentinel 並使用。
  - `04 panic/recover`：sample=使用時機（少用）+ defer 清理；hw=小練習。
- **idiom/陷阱**：早 return、不吞錯；`%w` 包裝；`errors.Is/As`；panic 只用於真正異常。
- **教學大綱**：1 error 值/if err 2 建立與包裝 3 Is/As/sentinel/自訂型別 4 panic/recover/defer。

## 009-套件與慣例（package、go 指令、目錄結構、命名）〔真 package 特例〕
- **目標**：資料夾=package、大小寫匯出、cmd/internal/pkg 慣例、命名與 fmt/vet。
- **作業對**（此主題用真子 package + 一個 main，可遷移 `_封存/greeting`）：
  - `01 跨包`：sample=`pricing` 子 package（匯出 `Total`、未匯出 `applyTax`）+ main 呼叫；hw=建小 package 並跨包呼叫。
  - `02 目錄結構`：教學為主；hw=把給定檔案歸類到 cmd/internal/pkg（判斷/擺放題）。
  - `03 命名/fmt/vet`：sample=慣例命名 + gofmt/vet 示範；hw=修正不符慣例的命名。
- **idiom/陷阱**：資料夾=package；大小寫匯出；`internal/` 限制；`MixedCaps`、短名、`ErrXxx`；gofmt 無爭論。
- **教學大綱**：1 package/匯出 2 import 路徑 3 目錄結構 4 命名/fmt/vet。
- **結構註記**：資料夾用 ASCII `009-packages`（見上方特例）。實際建法：跨包 `sample-01-crosspkg/`（main + `pricing/`）跑 `go run ./009-packages/sample-01-crosspkg`；命名慣例用單檔 `sample-02-命名慣例.go`；目錄結構（cmd/internal/pkg）併入教學。

## 010-併發基礎（goroutine、channel、WaitGroup、Mutex、select）
- **目標**：能讀懂電商後端的並發；只求基礎、安全。
- **作業對**：
  - `01 goroutine/WaitGroup`：sample=並行寄多筆訂單通知、`WaitGroup` 等全部完成；hw=並行算多商品小計。
  - `02 channel`：sample=worker 由 channel 收訂單處理、回傳結果；hw=用 channel 傳結果。
  - `03 Mutex`：sample=並發扣庫存用 `sync.Mutex` 保護；hw=用 Mutex 保護計數器。
  - `04 select`：sample=`select` 多 channel/timeout 前導（接 011）；hw=小練習。
- **idiom/陷阱**：goroutine 便宜但要管理；「用通訊共享記憶體」；`WaitGroup` Add/Done/Wait；`Lock`+`defer Unlock`；`go test -race`。
- **教學大綱**：1 goroutine/WaitGroup 2 channel 3 Mutex 4 select。

## 011-context（cancel、timeout、request-scoped value）
- **目標**：電商 web handler 幾乎必吃 `ctx`；取消、逾時、傳 request 範圍值。
- **作業對**：
  - `01 傳遞`：sample=`ctx context.Context` 當第一參數層層傳；hw=寫吃 ctx 的函式。
  - `02 timeout/cancel`：sample=`context.WithTimeout` + `ctx.Done()` 取消慢操作；hw=為操作加 timeout。
  - `03 value`：sample=`context.WithValue` 傳 requestID（謹慎用）；hw=傳一個 request 範圍值。
- **idiom/陷阱**：`ctx` 放第一參數、不存進 struct；`WithCancel/Timeout` 記得 `defer cancel()`；`Value` 只放 request-scoped。
- **教學大綱**：1 為何要 context 2 傳遞 3 cancel/timeout 4 value/慣例。
- **銜接**：接 010（select/Done）。

## 012-JSON與時間（encoding/json、struct tag、time.Time）

- **教學大綱**：1 Marshal/Unmarshal 2 struct tag/omitempty 3 time.Time 操作/格式化。
- **銜接**：接 005（struct tag）。
