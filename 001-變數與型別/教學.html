1. var 的四種宣告寫法

寫法範例說明
先宣告後賦值var name string賦值前是「零值」（這裡是空字串）
宣告同時賦值＋明寫型別var price int = 590最完整、最囉嗦
省略型別（自動推斷）var stock = 20Go 從右邊值推出型別
批量宣告var ( a = 1; b int = 2 )一組相關變數放一起

2. := 短變數宣告（函式內最常用）

category := "周邊配件" 等同「var ＋ 自動型別推斷」，但只能寫在函式裡面。

慣例函式內優先用 :=；型別能自動推斷就別多寫。這是 Go code 裡最常見的宣告方式。
陷阱:= 不能用在函式外（package 層級）。package 層級的變數/常數只能用 var / const。

3. 零值 zero value —— Go 沒有「未初始化的垃圾值」

只要宣告了沒給初值，Go 一律給一個明確的「零值」，不會是隨機記憶體：

型別零值
int / float640
string""（空字串）
boolfalse
指標 / slice / map / interfacenil
電商意義新建一個 Order、沒填的欄位自動是零值（數量 0、金額 0、已付款 false）——這通常正好是「合理的初始狀態」，是 Go 刻意的設計。

4. const 常數（編譯期就固定）

const taxRate = 0.05：執行期不能改，適合稅率、狀態碼、上限這種「寫死的值」。寫法和 var 類似（可明寫型別或自動推斷、可批量）。

慣例常數常宣告在檔案/函式的最上方，一眼就能看到這支程式有哪些「不會變的設定」。

5. scope 作用域：變數只活在它的 { } 內

在一對大括號裡宣告的變數，離開那個區塊就消失。sample 裡的 flashDiscount 只在 { } 內有效，外面再用會編譯失敗（undefined）。

陷阱：變數遮蔽 shadowing內層用 := 宣告一個「同名」變數，會蓋掉外層的那個（是兩個不同變數）。改到的是內層那個、外層沒變——這是 Go 常見 bug，之後 if err := 會再遇到。

B 型別、運算符與型別轉換

對應檔案：sample-02-型別轉換.go ｜ 跑：go run 001-變數與型別/sample-02-型別轉換.go

6. 基本型別

型別用途備註
int整數（數量、庫存、以「元」計的金額）大小看平台（64 位機器上是 64 位）
float64小數（折扣率、重量）Go 小數字面值預設就是 float64
string文字（名稱、SKU）UTF-8，唯讀不可變
bool真假（是否上架、是否 VIP）只有 true / false
電商大坑：金額別用 float！浮點數存不下精確的 0.1，累加會出現 0.30000000000000004 這種誤差，金流對帳會出事。實務慣例：金額用整數存最小單位（例如以「分」為單位，1500 = 15.00 元），或用 decimal 套件。sample 用 int 存「元」正是這個原因。

7. 運算符

算術 + - * / %、比較 == != > < >= <=、邏輯 && || !。其中 % 是取餘數，常用在整除判斷與分頁。

整數除法會截斷100 / 3 得到 33（不是 33.33），小數直接被丟掉。要小數請先轉 float64（見下節）。

8. 型別轉換：Go 一定要「顯式」轉

Go 不會自動幫你混算不同型別。int 和 float64 要一起算，必須手動轉：

subtotal := 1770             // int
rate := 0.85                 // float64
total := float64(subtotal) * rate   // 先把 int 轉成 float64 才能相乘
n := int(total)              // float64 轉回 int → 直接「截斷」小數，不是四捨五入
語法T(x) 就是「把 x 轉成型別 T」。這跟呼叫函式長得一樣，但 int(...) / float64(...) 是型別轉換。
補充數字↔字串不是用 string(123)（那是轉成 Unicode 字元！），要用標準庫 strconv（strconv.Itoa / strconv.Atoi）。這裡先知道有這回事即可。

C 用 iota 做列舉（enum）

對應檔案：sample-03-iota列舉.go ｜ 跑：go run 001-變數與型別/sample-03-iota列舉.go

9. const + iota

Go 沒有 enum 關鍵字。慣用做法：自訂一個整數型別 ＋ iota 自動編號，就得到一組有名字、有順序、型別安全的常數。

type OrderStatus int
const (
    Pending   OrderStatus = iota // 0 待付款 ← 只在這寫一次 iota
    Paid                         // 1 已付款 ← 右邊省略，自動沿用 "= iota"
    Shipped                      // 2 已出貨
    Completed                    // 3 已完成
    Cancelled                    // 4 已取消
)
iota 的精確定義它是「這一行在 const 區塊裡的索引（第幾行）」，每個 const 區塊開頭重設為 0，跟你「第幾次寫 iota」無關。關鍵規則：某行把右邊算式整個省略 → 自動沿用上一行寫法，而 iota 每行 +1。
陷阱下面 const7 是 2 不是 0——因為它在區塊第 2 行，前兩行沒寫 iota 也照樣在數：
const ( a = 5; b int = 6; const7 = iota ) → const7 == 2。

10. 加 String() → 印出來是名字不是數字

光有數字還不夠像 enum。幫型別加一個 String() 方法，印它時就會顯示中文：

func (s OrderStatus) String() string {
    names := []string{"待付款","已付款","已出貨","已完成","已取消"}
    if s < Pending || s > Cancelled { return "未知狀態" }
    return names[s]
}
為什麼會自動印中文？fmt 有個約定：型別只要「有 String() 方法」，用 %v 印它時就自動呼叫。%v 印名字、%d 印底層整數——兩者對照跑一次就懂。
前情提要這種「只看你有沒有某個方法」的機制，就是 interface 的雛形（Stringer 介面）。等學到主題 007 介面會整個展開，這裡先感受一下。
